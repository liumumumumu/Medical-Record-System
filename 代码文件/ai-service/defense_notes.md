# A 成员答辩讲稿与演示提纲

## 三分钟讲解

1. **任务**：我负责医疗 NLP 和 AI 服务，包括症状、医学术语、辅助诊断、病历生成和安全建议，不负责前端、数据库或 SpringBoot。
2. **两条模型链路**：课程核心是 T5 病历生成；BERT 只做独立的辅助诊断分类。两者输入用途、输出位置和评估指标都不同，不能把 BERT 分类说成病历生成。
3. **病历生成**：用 `Randeng-T5-77M-MultiTask-Chinese` 处理真实患者自述/医患对话到规范病历的 Seq2Seq 任务。部署版书面化对齐按原病例去重后使用 4,933 条 gold 真实对话；弱标注中的口语摘抄不再进入书面化微调。另做过 2,658 条高质量口语扰动专项实验，但因真实 dev 指标下降没有部署。模型生成主诉、现病史、既往史、辅助检查四段；其余字段由程序在不新增事实的前提下书面化并组装。
4. **辅助诊断**：核心五类由 BERT 微调判断，扩展到 20 类靠 9,620 条知识检索；规则层处理同义词、否定、数值和危险信号。BERT 官方 test macro-F1 0.8598，旧 TF-IDF 模型只作降级。
5. **安全**：T5 输出先做口语书面化和原始事实锚定；既往史、辅助检查、附件数值以用户输入为权威。随后通过结构、数值、医学术语和文本锚点守卫；漏段由同一 T5 带前缀重试，仍失败才标记模板兜底。BERT 结果只显示在 AI 辅助区，绝不覆盖正式病历。

## 演示顺序

1. 启动 `python app.py`，访问 `/health`，同时展示 `recordGeneratorBackend: transformer` / `recordGeneratorVersion: record-gen-t5-v1.2.0` 与辅助诊断的 `modelBackend: transformer` / `modelVersion: 2.0.0`。
2. 输入一个全是口语的病例，指出 T5 整理四个叙述段，程序继续规范过敏史、生命体征、诊断、治疗和用药；展示“肚子疼/拉肚子/吃过药”变成“腹痛/腹泻/曾服用”，同时数值和药名不变。
3. 展示结果页“Transformer 生成”徽标，并下载 DOCX：正式病历在前，空白医生签名在末尾，AI 辅助分析另起一页。
4. 输入“无高血压糖尿病史”，说明否定识别不会把这两项作为辅助诊断阳性依据。
5. 输入血压 `165/105mmHg`，展示数值规则；再输入持续胸痛，展示危险信号优先提示。
6. 展示病历生成的 test/六科室验收指标，再展示 BERT 与旧模型混淆矩阵；明确两组指标不可混用。

病历生成 v1.2.0 的独立 test 共 811 条：BLEU-2 0.3990、ROUGE-L 0.5357、四段解析/完整率 100%、数值一致率 98.40%、关键术语一致率 94.70%。BLEU-2 与数值一致率未达到旧门槛，要主动说明；运行时数值守卫会拒绝新增数值。5 组短口语专项验收全部走 Transformer、兜底 0、必需事实完整、指定口语残留 0；真实 HTTP 验收的摘要/结构化字段不一致 0、正文口语残留 0、DOCX 缺失 0。不要再展示旧版因目标泄露得到的 0.998 分数。


## 常见提问

**为什么生成病历用 T5，而诊断用 BERT？**

任务类型不同：BERT 是编码器，适合把整段文本压成类别概率；T5 是编码器—解码器 Seq2Seq 模型，适合把输入事实转换成有结构的病历文本。为控制幻觉，我们没有让 T5 自由生成整份文书，而只让它规范化四个叙述段，其余高风险事实由程序组装。

**为什么用 BERT？决策过程是什么？**

正式模型就是 BERT 微调（v2.0.0），但这是两轮实验后的综合决策，不是盲目上深度学习（`transformer_comparison.md`、`augmentation_report.md`）：第一轮只用 1,358 条金标数据，BERT 仅比 TF-IDF+逻辑回归基线高 0.67 pt，不值得付出 462 倍参数和 66 倍延迟；第二轮用远程监督从公开儿科问答扩充 2,500 条弱标签后，BERT 提升到 test macro-F1 0.8598–0.8698，最难的上感/支气管炎混淆也改善了，综合考虑准确率与扩展空间后正式替换，旧模型弃用（仅作无 GPU 环境的自动降级兜底，接口契约一致）。切换时还发现并修复了 softmax 过尖挤占扩展类 top1 的问题，用温度缩放 T=2.5 校准。同时实验证明：加成人内科数据反而变差（分布匹配 > 数据量）、弱标签过量会中毒、纯弱标签只有 0.66（一条金标≈五条弱标）。规则层与知识检索层不变，BERT 只替换融合公式里"监督模型分"这一路。

**20 类是不是都有 85% 准确率？**

不是。0.8598 只对应 IMCS-21 五类监督模型；其余十五类采用知识检索和规则，只报告课程样例 Top-3 命中率，不冒充临床准确率。

**为什么使用规则？**

规则用于否定词、数值指标、同义词和危险信号，这些逻辑需要确定性和可解释性；监督模型负责从整体文本学习分类边界，两者互补。

**如何避免危险建议？**

病历生成不把医生回答中的建议伪装成“患者已经接受的治疗”，诊断、治疗、用药权威段只采用前端输入。辅助建议不提供药物剂量，只输出一般观察、复查和就医分级，并统一添加免责声明。

**系统最大的不足是什么？**

训练语料偏儿科，扩展病种缺少独立临床测试；病历生成 test 又是结构字段到规范段落的受约束整理，不能把高文本相似度解释成临床有效性。因此系统定位始终是课程演示和辅助分析。
