# 多病种辅助分析模型报告

## 1. 任务与结论

本报告只说明**辅助诊断支链路**：模型输入患者主诉、现病史、既往史、体格检查和辅助检查，输出症状、医学术语、候选诊断、判断依据和安全建议。系统采用“监督分类 + 疾病知识检索 + 可解释规则”，用于课程演示而非临床诊断。正式病历由另一条独立的 Randeng-T5 Seq2Seq 链路生成，训练数据、事实守卫和验收指标见 `../../docs/project/record-generation-implementation.md`；BERT 的分类结果不会覆盖病历中的医生初步诊断。

**当前正式推理模型为 `medical-record-hybrid-diagnosis:2.0.0`（BERT 微调后端）**：`bert-base-chinese` 在 IMCS-21 官方划分金标 1,358 条 + 远程监督弱标签 2,500 条上微调，官方测试集 accuracy 0.8499、macro-F1 0.8598（同配方多次运行 0.8598–0.8698），推理时温度缩放 T=2.5 校准概率尺度。本报告第 2–5 节描述的 TF-IDF + 逻辑回归模型（v1.0.0，accuracy 0.8366、macro-F1 0.8491）是它的前身与对照基线——经两轮对比实验综合评估后已弃用，仅在无 BERT 权重或无 torch 的环境自动降级兜底。完整决策过程见 `transformer_comparison.md` 与 `augmentation_report.md`。

人工输入的 `preliminaryDiagnosis` 不进入模型文本，避免把前端已有结论当作模型证据。两代模型均超过项目设定的 0.80 验收线；20 类人工编写的自然语言验收样例 Top-3 命中率为 1.00（`models/curated_metrics.json`），该指标只验证本项目覆盖范围，不等同于临床准确率。

## 3. 数据处理

1. 从 IMCS-21 保留上呼吸道感染、小儿感冒、小儿支气管炎、小儿腹泻、小儿便秘，并将儿科标签规范化为普通感冒、支气管炎、腹泻、便秘。
2. 合并原始自述和报告中的主诉、现病史、既往史、辅助检查；不使用报告中的诊断和建议字段，避免标签泄漏。
3. 使用 2–5 字符 n-gram TF-IDF，`min_df=2`、`max_features=70000`，实际得到 44,238 个特征。
4. Logistic Regression 使用平衡类别权重和最多 1,500 次迭代，随机种子固定为 42。
5. 症状抽取执行同义词标准化、否定识别和体温/血压/血糖数值规则；知识检索使用 2–4 字符 n-gram 余弦相似度。

## 4. 融合策略

- 核心五类：监督模型 55% + 知识检索 30% + 规则 15%。
- 其余十五类：知识检索 70% + 规则 30%。
- 少于两个有效症状或最高融合分低于 0.25 时，`diagnosisTop1` 返回“暂无法确定”。
- 候选诊断最多三个，最低候选分为 0.15；诊断原因列出命中的阳性症状和已排除的否定症状。
- 危险症状不改变诊断分数，但会覆盖普通建议并优先提示急诊或及时就医。

## 6. 安全与限制

- 核心模型数据以儿科在线问诊为主，不能代表全部年龄和医疗场景。
- 扩展十五类是知识检索和规则结果，没有独立临床标注测试集；Top-3 样例指标不能作为医疗有效性证明。
- 症状文本无法替代医生查体、影像和实验室检查，输入不完整时必须允许“不确定”。
- 不使用知识库中的具体治疗文本，不生成药名、剂量或处方；建议只包含生活管理、复查和就医分级。
- 系统不记录真实患者信息，所有展示数据均为公开语料或虚构样例。

## 7. Transformer 升级（2026-07-16 已完成，v2.0.0 为正式模型）

本节原为升级设想，现已通过两轮对照实验落地并正式替换，完整数据见 `transformer_comparison.md` 与 `augmentation_report.md`：

- **v2.0.0（正式模型，BERT）**：`bert-base-chinese` 微调，训练集为 IMCS-21 官方 1,358 条金标 + 2,500 条远程监督弱标签（儿科问答语料），官方测试集 accuracy 0.8499、macro-F1 0.8598（同配方多次运行区间 0.8598–0.8698，全部高于 v1 基线）；推理时温度缩放 T=2.5，修复 softmax 过尖挤占扩展类 top1 的问题（40 条验收样例 top1 命中率 0.600→0.775）。
- **v1.0.0（已弃用）**：字符 TF-IDF + 逻辑回归，accuracy 0.8366、macro-F1 0.8491。综合评估（准确率、最难类改善、后续扩展空间）后弃用，仅作无 BERT 权重/无 torch 环境的自动降级兜底，保证仓库在任意机器可运行、测试可通过。
- **实现方式**：两代模型只替换融合公式中的"监督模型分"一路，知识检索层、规则层、接口契约与安全边界完全不变。
- 后续方向：独立外部测试集、多种子集成、医生评审、更充分的概率校准仍然成立。
