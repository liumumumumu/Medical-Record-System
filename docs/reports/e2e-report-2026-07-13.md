## 本轮修复（A / AI，已完成）

问题：Spring Boot 的 Flask 契约（`backend-service/docs/flask_contract.md`）要求 `/nlp/analyze` 成功响应包含 `model_version`（并支持可选 `content`），AI 服务未返回，导致结果页 `analysis.modelVersion` 恒为 `unknown`、`content` 恒为空。

修复：`AnalysisResult.to_dict()` 增加 `modelVersion`（取 `src/config.py` 的 `MODEL_VERSION=1.0.0`，后端 `@JsonAlias` 兼容 camelCase）和 `content`（与 `analyze_frontend` 一致，取诊断依据文本）。

改动文件（ai-service 本地无 git，需 LLY 同步到远端 LLY 分支）：

- `ai-service/src/schema.py`：`to_dict()` 增加两个字段。
- `ai-service/tests/test_api.py`：契约测试同步断言 `modelVersion == "1.0.0"`、`content == diagnosisReason`。
- `ai-service/api_document.md`：响应示例与字段表补充两个字段。

复验：AI 23 项测试通过；重启 Flask 后直连及经 Spring Boot 全链路均返回 `modelVersion: 1.0.0`、`content` 非空。
