## A / LLY：已修复并推送

### 1. 否定症状在逗号枚举中误判

- 原问题成立：`否认发热，咳嗽，乏力` 会把咳嗽、乏力识别为阳性。
- 已修复：否定作用域跨逗号延续；遇到“但有、存在、出现、伴有”等明确阳性转折后结束。
- 已增加回归测试，包括 `否认有发热`，避免把否定表达中的“有”误当成阳性转折。

### 2. 年龄小数被静默截断

- 原问题成立：`1.9` 会被 `int()` 转成 `1`。
- 已修复：仅接受整数、整数值浮点数和整数字符串；`1.9`、`"1.9"` 返回 400。

### 3. 既往史在兼容接口中被错误设为必填

- 原问题部分成立：通用 `PatientInput` 已允许缺省，但 `/nlp/analyze/frontend` 和 `/nlp/analyze/standardized` 仍强制必填。
- 已修复：两个接口均改为选填，缺省输出“未提供”。
- OpenAPI、接口文档和 TypeScript 交接类型已同步。

### 4. CYL 标准化接口

- 旧问题已在此前提交中解决：`POST /nlp/analyze/standardized` 已存在并有测试，不是当前待办。

## B / CYL：待 CYL 修改

### P1. 年龄小数被静默截断

- 实测 `parse_age(1.9) == 1`。

### P1. “没有”中的“有”导致否定误判

- `NEGATION_CUES` 同时含“没有”，`POSITIVE_CUES` 又含单字“有”。
- 实测 `没有发热但有咳嗽` 输出发热、咳嗽均为阳性。
- 应按词组和作用域处理，至少保证发热为否定、咳嗽为阳性。

- 当前 14 个 `data/processed/*` 文件显示修改，但 `git diff --ignore-space-at-eol --quiet` 为干净。

- 所有 `to_csv()` 应显式设置 `lineterminator="\n"`；JSON/文本写入应显式使用 LF。

- 当前在线链路是 React -> Spring Boot -> Flask -> MongoDB。

- 若老师要求“每次在线提交都经过数据处理”，B 与 C 需确定服务化接口或等价实现；若只要求展示离线清洗，可在答辩中明确说明。

- 同义词和医学资源分散在两个模块，长期会产生不同标准化结果。
