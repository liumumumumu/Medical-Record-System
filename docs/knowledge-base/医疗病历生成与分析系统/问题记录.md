## 问题详情

### 默认面板高度与字段数量冲突

- 问题描述：880×480 的默认面板无法稳定容纳六个必填字段、分组说明、标题和操作按钮。
- 原因分析：目标尺寸和信息量冲突，继续压缩会损害可读性与操作空间。
- 解决方案：默认只展示姓名、性别、年龄、主诉四项；其他字段通过“填写完整信息”展开；提交时如缺必填字段会自动展开并聚焦错误。
- 经验总结：先确定首屏必须完成的最小任务，再决定尺寸，不要用压缩字号和间距强行满足固定高度。

### 表单卡片嵌套导致视觉不统一

- 问题描述：外层主面板内部的每个字段分组再次使用大圆角、背景和边框，页面像卡片集合而不是工作台。
- 原因分析：把“独立模块”等同于“每个模块都需要卡片”。
- 解决方案：保留唯一主面板，内部使用分组标题、说明和分隔线建立层级。
- 经验总结：模块独立依赖信息层级和间距，不一定依赖容器和阴影。

### 标签切换只有视觉状态，没有真实页面

- 问题描述：原导航点击后只改变标签样式，内容仍停留在病例录入页。
- 原因分析：初期为静态演示，没有接入 React Router 页面结构。
- 解决方案：建立 `/upload` 和 `/results` 路由，使用 `NavLink` 关联地址与当前状态。
- 经验总结：导航状态应由路由驱动，避免维护一套与 URL 分离的假状态。

### 结果刷新后丢失

- 问题描述：生成记录只存在 React 内存状态时，刷新结果页会进入空状态。
- 解决方案：将最新生成记录同步写入 `sessionStorage`，在 App 初始化时恢复。
- 注意：这只是演示方案，真实系统应由后端保存，并处理权限、过期和脱敏。

### 医疗数据隐私与安全

- 问题描述：真实病例可能包含姓名、病史、检查结果等敏感信息。
- 风险：未经授权的收集、日志、存储或展示会造成严重隐私问题。
- 当前应对：课程演示只使用虚构或脱敏数据；页面明确结果不替代医师判断。
- 后续要求：明确数据最小化、传输加密、存储策略、访问控制、日志脱敏、删除机制和演示数据边界。
