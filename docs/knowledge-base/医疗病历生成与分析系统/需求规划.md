## 需求优先级

| 需求 | Must / Should / Could / Won't | 当前结论 |
| --- | --- | --- |
| 配置化字段、校验、生成入口、结果页 | Must | 前端已完成，继续联调后端 |
| 真实病例生成与分析接口 | Must | 下一阶段最高优先级 |
| 脱敏数据、隐私和医学免责声明 | Must | 演示与后端设计都必须保留 |
| 真实登录认证 | Should | 在需要多用户演示前完成 |
| 文件上传与解析 | Should | 视课程最终范围决定处理深度 |
| 动画与高级视觉反馈 | Could | 页面稳定后统一实现 |
| 移动端深度适配 | Could | 当前 PC 优先，只保证基础可用 |

## 用户场景

- 作为演示人员，我希望快速录入患者核心信息，以便展示病例生成流程。
- 作为演示人员，我希望展开补充完整病史、检查和诊疗信息，以便获得更完整的结果。
- 作为查看者，我希望在独立结果页阅读摘要、病历和分析，以便快速理解系统输出。
- 作为登录用户，我希望看到清晰的账户状态，以便理解系统具备用户体系扩展入口。

## 版本规划 / Roadmap

- v0.1：完成 PC 端导航、首页、880px 病例录入区域和字段配置。
- v0.2：完成真实路由、完整校验状态、结果页、演示登录和统一视觉规范。
- v0.3：定义并接入后端 API、真实认证、文件上传和分析结果。
- v0.4：完成动画、视觉精修、人工验收和自动化测试。
- v1.0：完成课程演示部署、发布材料和阶段复盘。

## 非目标

- 当前阶段不实现真实医学诊断决策，不替代执业医师。
- 当前阶段不优先设计复杂移动端体验。
- 动画不在页面和交互状态稳定之前开发。
- 不让 Ant Design 接管整体视觉，只使用少量图标或简单能力。

## 技术选型

- React 19 + TypeScript：组件和状态组织。
- Vite 7：开发与生产构建。
- React Router：录入页、结果页和兜底路由。
- Axios：预留后端请求，当前尚未实际接入。
- Ant Design Icons：少量图标。
- 自定义 CSS：主要 UI、版心、表单和响应式视觉。

## 架构设计

- `App.tsx` 负责公共页面壳、路由、登录演示状态和生成记录会话状态。
- `form-config.ts` 作为病例字段的配置来源。
- `upload-module.tsx` 负责动态渲染、校验、展开收起和提交。
- `results-page.tsx` 负责结果空状态与四区结果展示。
- `styles.css` 维护全局设计令牌和响应式规则。
