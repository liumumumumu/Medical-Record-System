## 开始工作

1. 如果仓库根目录有 `handoff-pending.md`，先将其同步回知识库；同步成功后删除它。
2. 读取项目上下文、任务池、未解决问题和最近两条工作记录。
3. 运行 `git status --short --branch`，再检查与当前任务相关的代码。
4. 简短说明当前阶段、当前任务、阻塞和准备执行的动作；没有冲突时直接继续当前任务。

## 检查点与收尾

每完成一个可验证工作单元，例如完成一个功能、做出关键决策、通过构建或发现新问题：

1. 在任务池更新完成项、当前任务和接下来任务。
2. 将新问题快速写入问题暂存。
3. 在工作记录追加目标、结果、验证、决策和下一步。
4. 只有阶段、阻塞或关键约束变化时才更新项目上下文。

如果无法写入知识库，在仓库根目录创建 `handoff-pending.md`，写明本轮完成内容、验证、任务变化和问题；不得静默结束。下次对话必须先同步该文件。

## 项目约束

- 前端目录是 `frontend/`；先在该目录读取实际脚本并运行命令。
- PC 优先；不恢复卡片套卡片的布局。
- 不把演示登录、sessionStorage 或字段拼接描述成真实生产能力。
- 仅使用虚构或脱敏病历；真实数据接入前先完成隐私、权限和存储边界。
- 先锁定 API 协议和完成 UI 验收，再追加动效。
